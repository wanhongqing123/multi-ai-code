# 设置弹窗 JSON 修复与截图快捷键预设设计

## 背景

当前“设置”弹窗已经同时承载两类配置：

- 项目级 AI CLI 配置，保存在每个项目目录下的 `project.json`
- 应用级截图快捷键配置，保存在应用根目录下的全局设置文件

最近出现了两个直接影响可用性的问题：

1. 某些项目的 `project.json` 已经被写坏，表现为一个完整 JSON 对象后面又拼接了额外字段文本。此时用户在设置弹窗里点击保存，会在主进程侧直接触发 `JSON.parse(...)` 异常，并把原始错误消息直接显示到界面上，例如：
   `Unexpected non-whitespace character after JSON at position 233`
2. 截图快捷键当前只支持纯文本手动输入。对大多数用户来说，这种交互门槛偏高，也更容易输错。

本设计要同时解决这两个问题，并保持现有“设置”弹窗的整体结构不变。

## 目标

- 在保存项目级 AI 设置时，自动修复可恢复的 `project.json`
- 修复后继续完成本次保存，而不是仅弹错退出
- 自动修复时保留原文件备份，避免误修后无法回退
- 将截图快捷键交互改为“预设组合优先，自定义输入兜底”
- 自定义输入框默认收起，仅在用户点击“自定义”后展开
- 保持截图快捷键仍由主进程实际注册结果决定是否生效

## 非目标

- 不实现通用 JSON 容错编辑器
- 不尝试修复括号缺失、字段名损坏、嵌套结构断裂等复杂 JSON 破坏
- 不实现快捷键录制器
- 不改变现有全局截图快捷键的主进程注册机制
- 不把截图快捷键改成项目级设置

## 现状

### 项目级设置保存链路

`project:set-ai-settings` 和 `project:set-repo-view-ai-settings` 当前都会：

1. 读取 `<project>/project.json`
2. 直接 `JSON.parse(raw)`
3. 修改 `ai_settings` 或 `repo_view_ai_settings`
4. 全量写回

这个实现默认 `project.json` 永远是合法 JSON。一旦文件尾部多出额外文本，保存链路会立即失败。

### 截图快捷键交互

当前截图快捷键区块包含：

- 启用开关
- 文本输入框
- 恢复默认按钮

它缺少常用预设，用户需要自己理解 Electron accelerator 语法并手动输入。

## 设计概览

本次设计分成两条独立但同时落地的改动：

1. 在主进程新增“项目元数据安全读取与自动修复”能力，接管 `project.json` 的读取入口
2. 在设置弹窗内将截图快捷键输入方式改为“预设按钮 + 自定义展开输入”

两条改动互不耦合：

- `project.json` 修复只影响项目级 AI 设置保存
- 快捷键预设只影响应用级截图快捷键的交互层

## 一、`project.json` 自动修复设计

### 适用范围

自动修复只覆盖一种保守且高概率的损坏形式：

- 文件前半段已经是一个完整、可解析的 JSON 对象
- 对象后面额外拼接了脏文本

例如：

```json
{
  "id": "p_xxx",
  "name": "demo"
}
"ai_settings": { "ai_cli": "claude" }
```

这类文件可以通过“截取第一个完整 JSON 对象”安全恢复。

### 不自动修复的情况

以下情况不做自动修复：

- 缺少闭合括号
- JSON 对象内部语法已经破坏
- 文件开头就不是对象
- 无法确定对象边界

这些场景直接返回明确错误，避免错误猜测造成二次污染。

### 主进程数据流

为 `project.json` 增加统一的安全读取函数，行为如下：

1. 读取原始文本
2. 尝试正常 `JSON.parse`
3. 若成功，直接返回对象
4. 若失败，进入“首个完整对象提取”流程
5. 若成功提取：
   - 将原文件备份到同目录，文件名追加时间戳和 `.bak`
   - 用提取出的合法 JSON 覆盖原文件
   - 返回解析后的对象，并标记“本次发生了自动修复”
6. 若提取失败：
   - 返回结构化错误
   - 不改原文件

后续 `project:set-ai-settings` 和 `project:set-repo-view-ai-settings` 都改为使用这个安全读取函数，而不是直接 `JSON.parse(raw)`。

### 对象提取策略

只提取“文件中的第一个顶层 JSON 对象”。实现原则：

- 从首个非空白字符开始，要求必须是 `{`
- 用逐字符扫描方式跟踪：
  - 花括号层级
  - 字符串状态
  - 转义状态
- 当顶层对象闭合时，截取 `[start, end]` 范围
- 截取结果再次 `JSON.parse` 验证

只有二次验证通过，才允许进入自动修复。

### 备份策略

自动修复前必须备份原文件。备份文件建议形式：

- `project.json.autofix-2026-05-20T10-22-33-123Z.bak`

约束：

- 备份文件与原文件同目录
- 仅在真的发生自动修复时创建
- 自动修复失败不生成空备份

### 用户可见反馈

自动修复成功且本次保存也成功时，不显示底部红错。改为轻量提示，例如 toast：

- `项目设置文件已自动修复并保存`

自动修复失败时，设置弹窗底部显示明确错误，例如：

- `项目设置文件已损坏，无法自动修复`

不再把原始 `JSON.parse` 技术错误直接暴露给用户。

## 二、截图快捷键预设交互设计

### 默认预设

固定提供 4 个常用组合：

- `CommandOrControl+Shift+A`
- `CommandOrControl+Shift+S`
- `CommandOrControl+Alt+A`
- `Alt+Shift+A`

这些预设直接对应用户可点击按钮，不要求用户手动输入。

### 自定义输入入口

在预设按钮旁提供一个 `自定义` 按钮：

- 默认不展开输入框
- 用户点击 `自定义` 后，才显示文本输入框
- 输入框仍接受标准 Electron accelerator 字符串

### 初始展示规则

打开设置弹窗时：

- 如果当前快捷键值属于 4 个预设之一：
  - 对应预设按钮高亮
  - 自定义输入框保持收起
- 如果当前快捷键值不属于预设：
  - 不改当前值
  - 自动展开自定义输入框
  - 让用户能直接看到当前配置

### 交互行为

- 点击某个预设按钮：
  - 立即把当前快捷键值切换为该组合
  - 高亮该按钮
  - 收起自定义输入框
- 点击 `自定义`：
  - 展开输入框
  - 保留当前值
- 点击 `恢复默认`：
  - 启用截图快捷键
  - 当前值恢复到 `CommandOrControl+Shift+A`
  - 高亮默认预设
  - 收起自定义输入框

### 禁用状态

当“启用全局截图快捷键”关闭时：

- 预设按钮置灰
- `自定义` 按钮置灰
- 如果自定义输入框已展开，也一并置灰
- 当前值保留，不清空

## 三、保存与错误处理

### 保存顺序

设置弹窗点击保存时：

1. 先提交应用级截图快捷键设置
2. 再提交项目级 AI 设置

这个顺序保持现状不变。

### 项目级设置的错误语义

项目级设置保存返回值改为结构化错误，不再直接透传原始 `JSON.parse` 文本。建议语义：

- `project settings auto-repaired`
- `project settings corrupted and unrecoverable`

渲染层再将其转换成用户友好的中文提示。

### 快捷键保存逻辑

截图快捷键保存逻辑保持现状：

- 主进程尝试注册
- 成功才落盘
- 失败则回滚到旧值

预设按钮不会绕过这条主进程校验链路。

## 四、组件与模块改动

### 主进程

- `electron/main.ts`
  - 抽出项目元数据安全读取与自动修复逻辑
  - 更新 `project:set-ai-settings`
  - 更新 `project:set-repo-view-ai-settings`

如有必要，可新增小型辅助模块，例如：

- `electron/store/projectMeta.ts`

职责仅包括：

- 安全读取 `project.json`
- 自动修复可恢复损坏
- 生成备份

### 渲染层

- `src/components/AiSettingsDialog.tsx`
  - 新增截图快捷键预设按钮组
  - 新增 `自定义` 展开态
  - 调整错误文案展示

### 样式

- `src/styles.css`
  - 为预设按钮组增加布局
  - 为选中态、置灰态、自定义展开区增加样式

## 五、测试设计

### 主进程测试

新增或补充以下自动化测试：

1. 合法 `project.json` 可正常读取并保存
2. `project.json` 尾部有脏文本时，会自动修复并继续保存
3. 自动修复时会生成备份文件
4. 无法恢复的坏 JSON 返回明确错误，且不覆盖原文件
5. 自动修复成功后，本次设置写入结果正确

### 前端测试

为设置弹窗补充以下测试：

1. 当前值属于预设时，对应按钮高亮，自定义输入框默认收起
2. 点击某个预设按钮后，值切换且自定义输入框收起
3. 点击 `自定义` 后才显示输入框
4. 当前值不属于预设时，自定义输入框默认展开
5. 点击 `恢复默认` 后恢复默认值并收起自定义输入框
6. 禁用截图快捷键时，预设和自定义按钮都置灰

### 手动验证

1. 使用一份尾部拼接脏文本的 `project.json` 打开设置并保存
   - 预期：保存成功，文件被修复，并生成备份
2. 将截图快捷键切到某个预设并保存
   - 预期：保存成功，快捷键立即生效
3. 点击 `自定义`，输入非预设组合并保存
   - 预期：若组合合法则生效，若冲突则保留旧值并提示失败
4. 关闭截图快捷键并保存
   - 预期：预设区置灰，主进程不再注册快捷键

## 决策记录

| 决策 | 选项 | 选定 | 理由 |
|------|------|------|------|
| `project.json` 处理方式 | 仅提示 / 自动修复 / 二次确认后修复 | 自动修复 | 用户明确要求自动修复并继续保存 |
| 自动修复范围 | 通用 JSON 修复 / 保守提取首个对象 | 保守提取首个对象 | 风险更低，正好覆盖当前故障 |
| 快捷键主交互 | 纯输入框 / 下拉 / 预设按钮 | 预设按钮 | 点击成本最低，最适合常用组合 |
| 自定义输入展示 | 常显 / 点按钮展开 | 点按钮展开 | 降低视觉噪音，同时保留高级入口 |
