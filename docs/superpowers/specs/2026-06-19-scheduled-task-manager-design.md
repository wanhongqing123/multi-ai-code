# 定时任务管理模块设计

## 背景

Multi-AI Code 当前已经有一个主 AICLI 会话，用户可以在主界面中与 AI 交互，也可以通过 Skill 编排等功能把 prompt 发送给当前 AICLI。

新需求是增加一个“定时任务管理模块”：用户可以创建任务，配置任务在什么时间执行，并在到达时间后把任务交给当前 AICLI 处理。

本设计采用“只使用当前 AICLI”的方案，不为每个任务创建新的 AICLI 进程。这样可以避免多个 AICLI 同时竞争 Claude 登录态、配置目录、权限确认和上下文资源。

## 已确认方向

- 定时任务使用当前 AICLI 会话执行。
- AICLI 忙碌时，定时任务默认排队等待，不直接打断用户当前对话。
- 用户创建任务时，需要明确配置：
  - 任务名称。
  - 任务说明。
  - AICLI 要做什么。
  - AICLI 怎么做。
  - 哪些行为禁止。
  - 执行时间。
  - 最终发送给 AICLI 的 prompt 预览。
- 第一版先做清晰、可用、可控的任务管理，不做多 AICLI 并发。

## 目标

- 在顶部工具栏增加 `定时任务` 入口。
- 提供定时任务管理弹窗。
- 提供新建、编辑、删除、启用、禁用任务能力。
- 支持查看任务状态、下次执行时间、最近执行结果。
- 支持创建任务时配置 AICLI 要执行的具体内容。
- 支持保存前预览最终发送给 AICLI 的 prompt。
- 到达任务执行时间后，将任务放入 AI 工作队列。
- 队列在当前 AICLI 空闲时，通过受控入口发送任务 prompt。

## 非目标

- 第一版不支持多个 AICLI 并发执行。
- 第一版不支持 cron 表达式编辑器。
- 第一版不支持复杂日历视图。
- 第一版不支持任务之间的依赖编排。
- 第一版不自动提交 git。
- 第一版不默认允许定时任务自动修改代码。
- 第一版不隐藏任务 prompt，用户必须能看到最终发送内容。

## 用户入口

顶部工具栏新增按钮：

```text
⏰ 定时任务
```

建议位置：

```text
设置 | 习惯监控 | Skill 管理 | 定时任务 | 截图采样 | 重置主会话
```

点击后打开 `定时任务管理` 弹窗。

## 任务管理界面

任务管理弹窗采用左右两栏布局。

### 顶部区域

顶部展示模块标题和当前 AICLI 状态：

```text
⏰ 定时任务管理
当前 AICLI：空闲 / 忙碌 / 未启动
```

然后展示统计卡片：

- 已启用任务数量。
- 排队中任务数量。
- 运行中任务数量。
- 当前 AICLI 状态。

### 操作栏

操作栏包含：

- `+ 新建任务`
- `全部启用`
- `全部禁用`
- `刷新`
- 搜索框

搜索范围包括任务名称、任务说明、任务状态和任务 prompt 内容。

### 左侧任务列表

每个任务以卡片展示：

- 任务名称。
- 启用状态。
- 当前状态。
- 简短说明。
- 下次执行时间。
- 最近执行结果。
- `运行` 按钮。
- `编辑` 按钮。

状态颜色：

- 启用：绿色。
- 排队：橙色。
- 禁用：灰色。
- 运行中：蓝色。
- 失败：红色。

### 右侧详情区

选中任务后，右侧展示：

- 任务内容。
- 执行计划。
- AICLI 策略。
- 最近运行记录。

AICLI 策略固定展示：

```text
使用当前 AICLI
忙碌时排队等待
```

## 创建任务界面

点击 `+ 新建任务` 后打开新建任务弹窗。

该界面不是简单填写时间，而是帮助用户完整描述“让 AICLI 做什么、怎么做、哪些事不能做”。

### 左侧步骤区

左侧展示配置步骤：

1. 基本信息。
2. AICLI 要做什么。
3. 怎么干与限制。
4. 执行时间。
5. Prompt 预览。

左侧还展示默认安全策略：

- 默认不允许自动改代码。
- 默认不允许自动提交。
- AICLI 忙时排队等待。

### 中间配置区

中间是主要表单。

字段：

- 任务名称。
- 任务说明。
- 让 AICLI 做什么。
- 具体要求。
- 执行方式。
- 执行时间。
- 超时时间。

`具体要求` 提供常用复选项：

- 分析代码风险。
- 给出修改建议。
- 不要直接修改代码。
- 运行测试前先说明。

第一版默认勾选：

- 分析代码风险。
- 给出修改建议。
- 不要直接修改代码。

第一版默认不勾选：

- 运行测试前先说明。

执行方式第一版固定为：

```text
使用当前 AICLI
忙碌时排队等待
```

### 右侧 Prompt 预览区

右侧展示最终会发送给 AICLI 的内容。

示例：

```text
你现在要执行一个定时任务。

任务名称：每日代码巡检
工作目录：E:\OpenSource\multi-ai-code

任务目标：
检查当前项目最近的代码变更，找出潜在风险。

执行要求：
1. 分析代码风险。
2. 给出修改建议。
3. 不要直接修改代码。
4. 不要提交 git。

输出要求：
1. 用中文总结。
2. 明确列出发现的问题。
3. 明确列出建议的下一步。
```

用户保存前必须能看到该 prompt，避免“定时任务背后偷偷让 AI 做事”。

## 任务状态

任务本身有启用状态：

- `enabled`
- `disabled`

任务运行记录有执行状态：

- `pending`：等待执行。
- `queued`：已到时间，等待 AICLI 空闲。
- `running`：已发送给当前 AICLI。
- `succeeded`：执行完成。
- `failed`：执行失败。
- `timed_out`：执行超时。
- `cancelled`：用户取消。
- `skipped`：错过执行且策略为跳过。

## 调度规则

定时任务调度器负责扫描到期任务。

流程：

1. 周期性扫描已启用任务。
2. 找到到期任务。
3. 创建一条运行记录。
4. 放入 AI 工作队列。
5. 如果当前 AICLI 未启动，任务保持 `queued`，并提示用户启动 AICLI。
6. 如果当前 AICLI 忙，任务保持 `queued`。
7. 如果当前 AICLI 空闲，发送任务 prompt。
8. 发送成功后状态变为 `running`。
9. 根据输出捕获、超时或失败事件更新运行记录。

## AICLI 使用策略

第一版只使用当前主会话。

发送入口应使用受控接口，而不是直接写 PTY 原始输入：

```text
cc.sendUser(sessionId, prompt)
```

原因：

- 这个入口可以统一追加 Skill 策略上下文。
- 这个入口可以被测试。
- 这个入口比直接 `cc:input` 更容易管理发送来源。

定时任务发送时，prompt 应明确标记来源：

```text
你现在要执行一个由 Multi-AI Code 触发的定时任务。
```

这样用户在主会话里看到输出时，可以知道这不是普通手动输入。

## AI 工作队列

需要增加一个面向 AICLI 的工作队列。

队列职责：

- 保证同一时间只有一个自动任务占用当前 AICLI。
- 跟踪当前任务。
- 跟踪等待任务。
- 支持取消等待中的任务。
- 支持任务超时。
- 支持 UI 查询队列状态。

队列不负责业务配置。它只负责“什么时候可以把一段 prompt 发给 AICLI”。

## 数据模型

建议增加两类数据。

### scheduled_tasks

保存任务配置。

核心字段：

- `id`
- `project_id`
- `name`
- `description`
- `enabled`
- `goal`
- `instructions`
- `allow_code_changes`
- `allow_git_commit`
- `require_test_confirmation`
- `schedule_type`
- `schedule_time`
- `schedule_days`
- `timeout_minutes`
- `created_at`
- `updated_at`

### scheduled_task_runs

保存每次运行记录。

核心字段：

- `id`
- `task_id`
- `status`
- `scheduled_at`
- `started_at`
- `finished_at`
- `prompt`
- `output_excerpt`
- `error`
- `timeout_minutes`

## 错误处理

### 当前 AICLI 未启动

任务进入 `queued`。

UI 展示：

```text
等待 AICLI 启动
```

用户启动主会话后，队列继续执行。

### 当前 AICLI 忙碌

任务进入 `queued`。

UI 展示：

```text
AICLI 忙碌，排队等待
```

### 发送失败

运行记录变为 `failed`。

UI 展示失败原因，并提供手动重试。

### 执行超时

运行记录变为 `timed_out`。

第一版只标记超时，不强行杀掉当前 AICLI，因为当前 AICLI 是用户主会话，强行中断可能破坏用户上下文。

## 测试策略

### 单元测试

- 任务下次执行时间计算。
- prompt 生成逻辑。
- 任务状态转换。
- 队列串行执行。
- AICLI 未启动时的排队行为。
- 超时时的状态更新。

### 组件测试

- 顶部按钮是否渲染。
- 任务管理弹窗是否渲染。
- 新建任务弹窗是否渲染。
- Prompt 预览是否随表单变化。
- 启用/禁用按钮状态是否正确。

### 集成测试

- 创建任务后能在列表中出现。
- 到期任务会进入队列。
- 队列空闲时会调用 `cc.sendUser`。
- AICLI 未启动时不会丢失任务。

## 第一版交付范围

第一版实现：

- 顶部 `定时任务` 入口。
- 任务管理弹窗。
- 新建/编辑任务弹窗。
- 本地持久化任务配置。
- 本地持久化运行记录。
- Prompt 预览。
- AI 工作队列。
- 到期任务发送给当前 AICLI。
- 基础状态展示。

第一版不实现：

- 多 AICLI。
- cron 表达式。
- 任务依赖编排。
- 复杂可视日历。
- 自动修改代码权限细分到文件级。
- 后台隐藏执行。

## 后续扩展

后续可以扩展：

- 后台一次性 AICLI 执行模式。
- 多 AICLI worker 池。
- 任务模板。
- 任务运行报告。
- 与 Skill 管理联动，为某个任务限定可用 Skill。
- 与 Skill 编排联动，让定时任务触发某条 Skill 流水线。
